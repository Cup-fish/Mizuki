## 项目背景

为连锁门店打造的数字化运营看板，帮助管理层直观掌握门店经营状况。设计目标是 **1920×1080** 大屏展示，同时兼容不同设备分辨率。

## 技术栈

- **前端框架**：Vue 3 + Composition API + Vite
- **可视化**：ECharts 6, echarts-gl（3D 图表）
- **数据请求**：Axios
- **样式方案**：CSS 级联体系
- **AI 辅助**：CodeBuddy / Codex 加速图表组件开发

## 核心功能

### 多维度数据可视化

覆盖门店经营的六大分析维度：

- **客源画像分析** —— 综合展示顾客来源渠道分布
- **设备运营监控** —— 实时设备状态与告警展示
- **会员分析** —— 会员增长、活跃度、消费偏好
- **活动运营** —— 营销活动 ROI 与转化漏斗
- **业态分布** —— 不同业态门店的业绩对比
- **业务趋势** —— 时间维度的营收与客流趋势

### 复杂图表实现

综合运用多种图表类型，确保数据展示直观准确：

| 图表类型 | 应用场景 |
|----------|----------|
| 动态流向图 | 客源城市 → 门店流量分布 |
| 矩阵分布图 | 不同业态 × 区域交叉分析 |
| 三维对比图 (echarts-gl) | 多维度 KPI 立体对比 |
| 组合图表 | 趋势 + 柱状 + 标注联动 |

### 多设备适配

针对不同分辨率设备（大屏 / 桌面 / 平板 / 手机）采用差异化数据密度策略，在各类展示场景下保持稳定表现。

### 容错与加载态

设计 **Loading / Empty / Error** 三种状态的数据渲染策略，在接口异常时不影响整体页面展示，保证系统鲁棒性。

## 项目收获

- 深刻理解了 **数据可视化 ≠ 画图表**，关键在于如何选取正确的图表类型来回答业务问题
- 多数据源整合时需要做好异常隔离，单一接口失败不应影响其他模块
- AI 辅助开发在图表配置这类模板化工作中效率提升明显，交互效率提升约 **40%**
