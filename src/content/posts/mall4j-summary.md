## 项目背景

2025 年暑假参与了 mall4j 电商项目的开发工作，这是一个基于 Spring Boot + Vue 的前后端分离电商系统，涵盖商品管理、订单流程、用户权限等核心模块。

断断续续写了两个多月，踩了不少坑，这里做个复盘记录。

---

## 技术栈

- **后端**：Spring Boot 2.x、MyBatis-Plus、Redis、RabbitMQ
- **前端**：Vue 3、Element Plus
- **数据库**：MySQL 8.0
- **部署**：Docker + Nginx

---

## 做了哪些模块

### 商品模块

商品的 SPU / SKU 设计是整个系统最复杂的部分，一个 SPU 对应多个 SKU，每个 SKU 有独立的库存和价格。

最开始我对这个设计理解不深，导致接口写出来逻辑乱了一团。后来重新梳理了数据表关系，才弄清楚：

```
spu (商品基本信息)
 └── sku (规格组合 = 颜色 × 尺寸 × ...)
      └── sku_stock (库存)
```

### 订单模块

订单状态机是另一个难点。从「待付款」到「已完成」中间有很多状态流转，加上超时自动取消的逻辑，需要配合 Redis 的延迟队列来做。

用 RabbitMQ 的死信队列实现了超时取消，测试了一下，30 分钟未付款的订单确实会自动关闭，感觉还挺有成就感的。

### 权限模块

基于 RBAC 模型实现了用户-角色-权限三级管理，用 Spring Security + JWT 做鉴权。

这块相对来说参考资料比较多，踩的坑少一些，主要是 token 刷新机制花了点时间理清楚。

---

## 遇到的问题

**1. N+1 查询问题**

商品列表接口上线后发现响应很慢，排查之后发现是 MyBatis-Plus 在循环里触发了大量单条查询。

用 `@TableField` 配合 `association` 改成联表查询之后，接口响应从 800ms 降到了 120ms 左右。

**2. 并发超卖**

压测时发现库存扣减会出现超卖，最后用 Redis + Lua 脚本实现了原子性库存扣减，解决了并发问题。
